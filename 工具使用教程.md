## 📖 详细使用说明

### 1. scan-logging-issues.py

**功能：** 扫描所有 Java 文件，生成详细问题报告

**使用方法：**
```bash
python3 scripts/scan-logging-issues.py
```

**输出：**
- 问题总数统计
- 按文件分类统计
- 详细问题列表（前 100 条）
- 修复建议

**输出文件：** `logs_detailed_report_YYYYMMDD_HHMMSS.txt`

---

### 2. fix-logging-batch.py

**功能：** 批量修复日志问题

**使用方法：**
```bash
python3 scripts/fix-logging-batch.py
```

**处理模式：**
- 自动添加 Logger 导入
- 自动添加 Logger 声明
- 替换 System.out.println
- 替换 printStackTrace

**修复率：** 约 80%

---

### 3. fix-top10.py

**功能：** 修复问题最多的前 10 个文件

## 🔧 自定义配置

### 修改修复模式

编辑脚本中的 patterns 数组：

```python
patterns = [
    (r'System\.out\.println\("([^"]+)"\);', r'logger.info("\1");'),
    (r'System\.out\.println\("([^"]+)" \+ (\w+)\);', r'logger.info("\1: {}", \2);'),
    # 添加自定义模式
]
```

### 添加 Logger 声明

编辑脚本中的 logger 声明模板：

```python
logger_decl = f'\n\tprivate static final Logger logger = LoggerFactory.getLogger({class_name}.class);'
```

## ❓ 常见问题

### Q1: 工具可以用于其他项目吗？

**A:** 可以！工具是通用的，适用于所有 Java 项目。

**使用方法：**
```bash
# 修改脚本中的 src_dir
src_dir = '/path/to/your/project/src'

# 运行工具
python3 scripts/fix-logging-batch.py
```

---

### Q2: 修复后需要手动审查吗？

**A:** 建议审查核心业务代码。

**审查重点：**
- 日志文案是否清晰
- 日志级别是否合理
- 是否包含敏感信息
- 业务上下文是否完整

---

### Q3: 如何处理修复失败的文件？

**A:** 手动修复或调整脚本正则表达式。

**步骤：**
1. 查看修复失败的错误信息
2. 分析文件特殊格式
3. 添加对应的正则模式
4. 重新运行脚本

---

### Q4: 工具会修改注释代码吗？

**A:** 不会。工具会跳过注释中的 System.out.println。
