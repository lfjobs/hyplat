# 📖 Java 日志规范化使用指南

**适用范围:** Hyplat JavaWeb 项目及所有 Java 项目

## 🎯 日志规范目标

1. **统一日志格式** - 所有日志使用 SLF4J
2. **规范日志级别** - 正确使用 ERROR/WARN/INFO/DEBUG
3. **提升可读性** - 添加模块标签和业务上下文
4. **便于调试** - 结构化日志，快速定位问题
5. **安全合规** - 避免敏感信息泄露

---

## 📋 日志级别使用标准

### ERROR - 错误级别

**使用场景：**
- 系统错误、异常
- 业务逻辑失败
- 外部服务调用失败
- 需要立即处理的问题

**示例：**
```java
try {
    // 业务逻辑
    paymentService.pay(orderId, amount);
} catch (Exception e) {
    logger.error("[支付] 订单支付失败，订单号：{}, 金额：{}", orderId, amount, e);
}
```

**注意事项：**
- ✅ 必须记录完整堆栈信息（传入异常对象）
- ✅ 包含关键业务参数
- ❌ 不要捕获异常后不记录日志

---

### WARN - 警告级别

**使用场景：**
- 可恢复的错误
- 非预期的业务情况
- 配置项缺失（有默认值）
- 性能警告

**示例：**
```java
if (inventory < orderQuantity) {
    logger.warn("[库存] 库存不足，商品 ID: {}, 当前库存：{}, 订单数量：{}", 
                productId, inventory, orderQuantity);
    // 使用备用方案
}
```

**注意事项：**
- ✅ 说明警告原因
- ✅ 提供解决建议或备用方案
- ❌ 不要滥用警告级别

---

### INFO - 信息级别

**使用场景：**
- 关键业务流程
- 重要状态变更
- 系统启动/停止
- 用户关键操作

**示例：**
```java
logger.info("[订单] 订单创建成功，订单号：{}, 用户 ID: {}, 金额：{}", 
            orderId, userId, amount);

logger.info("[用户] 用户登录成功，用户 ID: {}, IP: {}", userId, ipAddress);
```

**注意事项：**
- ✅ 记录关键业务节点
- ✅ 包含唯一标识（订单号、用户 ID 等）
- ❌ 不要记录过于频繁的操作

---

### DEBUG - 调试级别

**使用场景：**
- 详细流程跟踪
- 参数值记录
- 中间状态
- 开发调试信息

**示例：**
```java
logger.debug("[查询] 执行 SQL: {}, 参数：{}", sql, params);
logger.debug("[缓存] 缓存命中，key: {}", cacheKey);
```

**注意事项：**
- ✅ 用于开发调试
- ✅ 生产环境可关闭
- ❌ 不要记录敏感信息

## 📝 日志文案规范

### 标准格式

```
[模块名] 操作描述，关键参数：{}
```

### 模块标签示例

| 模块 | 标签 | 示例 |
|------|------|------|
| 支付 | `[支付]` | `[支付] 订单支付成功` |
| 订单 | `[订单]` | `[订单] 订单创建成功` |
| 用户 | `[用户]` | `[用户] 用户登录成功` |
| 库存 | `[库存]` | `[库存] 库存扣减成功` |
| 物流 | `[物流]` | `[物流] 发货成功` |
| 数据库 | `[数据库]` | `[数据库] 查询成功` |
| 缓存 | `[缓存]` | `[缓存] 缓存更新成功` |
| 文件 | `[文件]` | `[文件] 文件上传成功` |
| 认证 | `[认证]` | `[认证] 权限验证通过` |
| 接口 | `[接口]` | `[接口] 外部调用成功` |

### 文案要求

**✅ 正确示例：**
```java
// 包含模块标签、操作描述、关键参数
logger.info("[支付] 订单支付成功，订单号：{}, 金额：{}, 支付方式：{}", 
            orderId, amount, paymentMethod);

logger.error("[数据库] 查询失败，SQL: {}, 错误：{}", sql, e.getMessage(), e);
```

**❌ 错误示例：**
```java
// 缺少模块标签
logger.info("支付成功");

// 缺少关键参数
logger.info("订单支付成功");

// 过于简单
logger.info("成功");

// 使用 System.out.println
System.out.println("支付成功");
```

## 🔒 安全规范

### 禁止记录的信息

**绝对禁止：**
- ❌ 用户密码（明文或加密）
- ❌ 银行卡号（完整）
- ❌ 身份证号（完整）
- ❌ 手机号（完整）
- ❌ API 密钥、Token
- ❌ 数据库连接密码

**脱敏处理：**
```java
// ✅ 正确：脱敏处理
String maskedPhone = phone.replaceAll("(\\d{3})\\d{4}(\\d{4})", "$1****$2");
logger.info("[用户] 手机号：{}", maskedPhone);

String maskedIdCard = idCard.replaceAll("(\\d{6})\\d{8}(\\d{4})", "$1********$2");
logger.info("[用户] 身份证号：{}", maskedIdCard);

String maskedBankCard = bankCard.replaceAll("(\\d{4})\\d{8}(\\d{4})", "$1********$2");
logger.info("[支付] 银行卡号：{}", maskedBankCard);
```

## 💡 最佳实践

### 1. 使用参数化日志

**✅ 正确：**
```java
logger.info("[订单] 订单创建成功，订单号：{}, 用户 ID: {}", orderId, userId);
```

**❌ 错误：**
```java
logger.info("[订单] 订单创建成功，订单号：" + orderId + ", 用户 ID: " + userId);
```

**原因：** 参数化日志在日志级别不输出时不会进行字符串拼接，性能更好。

---

### 2. 条件记录日志

**✅ 正确：**
```java
if (logger.isDebugEnabled()) {
    logger.debug("[调试] 详细数据：{}", complexData);
}
```

**原因：** 避免不必要的对象 toString() 调用。

---

### 3. 异常处理规范

**✅ 正确：**
```java
try {
    // 业务逻辑
} catch (BusinessException e) {
    logger.error("[业务] 业务异常，订单号：{}", orderId, e);
    throw e;
} catch (Exception e) {
    logger.error("[系统] 系统异常，订单号：{}", orderId, e);
    throw new SystemException("系统异常", e);
}
```

**❌ 错误：**
```java
try {
    // 业务逻辑
} catch (Exception e) {
    e.printStackTrace();  // ❌ 禁止使用
}
```

---

### 4. Logger 声明规范

**✅ 正确：**
```java
public class OrderService {
    private static final Logger logger = LoggerFactory.getLogger(OrderService.class);
    // ...
}
```

**❌ 错误：**
```java
// 非 static final
private Logger logger = LoggerFactory.getLogger(OrderService.class);

// 使用类名字符串
private static final Logger logger = LoggerFactory.getLogger("OrderService");
```

## 🛠️ IDE 配置

### IntelliJ IDEA

#### 1. 禁止 System.out.println

**步骤：**
1. File → Settings → Editor → Inspections
2. Java → System output
3. 勾选 "System.out.println() usage"
4. Severity 设置为 "Error"

#### 2. 配置 Live Templates

**步骤：**
1. File → Settings → Editor → Live Templates
2. 添加以下模板：

| Abbreviation | Template | Description |
|--------------|----------|-------------|
| loginfo | `logger.info("$PARAM$", $PARAM$);` | INFO 日志 |
| logerror | `logger.error("$PARAM$", $PARAM$);` | ERROR 日志 |
| logwarn | `logger.warn("$PARAM$", $PARAM$);` | WARN 日志 |
| logdebug | `logger.debug("$PARAM$", $PARAM$);` | DEBUG 日志 |
| logger | `private static final Logger logger = LoggerFactory.getLogger($CLASS$.class);` | Logger 声明 |

---

### Eclipse

#### 1. 配置 Save Actions

**步骤：**
1. Window → Preferences → Java → Editor → Save Actions
2. 勾选 "Perform the selected actions"
3. 配置代码格式化规则

#### 2. 配置代码检查

**步骤：**
1. Window → Preferences → Java → Compiler → Errors/Warnings
2. 搜索 "System.out"
3. 设置为 "Error"

## 🎯 常见问题

### Q1: 什么时候用 ERROR，什么时候用 WARN？

**A:** 
- **ERROR**: 系统错误，需要立即处理，影响业务
- **WARN**: 警告信息，不影响系统运行，但需要注意

### Q2: 日志级别如何配置？

**A:**
- **开发环境**: DEBUG
- **测试环境**: INFO
- **生产环境**: INFO 或 WARN
- **排查问题**: 临时调整为 DEBUG

### Q3: 如何避免日志过多？

**A:**
- 只记录关键业务节点
- 调试信息使用 DEBUG 级别
- 避免在循环中记录 INFO 日志
- 使用采样日志（如每 100 次记录一次）
