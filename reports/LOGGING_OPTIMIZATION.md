## 📋 优化规范

### 日志格式标准

**推荐格式：**
```
[模块名] 操作描述，关键参数：{}
```

**示例：**
```java
logger.info("[支付宝] 订单创建成功，订单号：{}, 金额：{}", orderId, amount);
logger.error("[数据库] 查询失败，SQL: {}, 错误：{}", sql, e.getMessage(), e);
logger.debug("[缓存] 缓存命中，key: {}", cacheKey);
```

---

### 模块标签建议

| 模块 | 标签 |
|------|------|
| 支付相关 | `[支付宝]` `[微信]` `[银联]` |
| 车辆管理 | `[车辆检测]` `[道闸控制]` `[费用结算]` |
| 预算管理 | `[预算管理]` `[支出]` `[收入]` |
| 数据库 | `[数据库]` `[SQL]` `[缓存]` |
| 文件操作 | `[文件]` `[上传]` `[下载]` |
| 用户认证 | `[认证]` `[授权]` `[登录]` |

---

### 日志级别使用指南

| 级别 | 用途 | 示例 |
|------|------|------|
| **ERROR** | 系统错误、异常 | `logger.error("支付失败", e)` |
| **WARN** | 警告、可恢复错误 | `logger.warn("库存不足")` |
| **INFO** | 关键业务流程 | `logger.info("订单创建成功")` |
| **DEBUG** | 调试信息、详细流程 | `logger.debug("查询参数：{}", params)` |
| **TRACE** | 最详细跟踪 | `logger.trace("方法入口")` |
